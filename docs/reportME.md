Описание предметной области
Система учёта записи учеников на уроки музыки в частной школе. Каждый преподаватель ведёт один или несколько инструментов. Ученик может записываться к разным преподавателям, но не более одного урока в день по одному инструменту. Реализована таблица домашних заданий с датами выдачи и дедлайнами.

Стек технологий
- СУБД: MySQL 5.7
- Кодировка: utf8mb4
- Инструменты управления: phpMyAdmin (хостинг Beget), MySQL Workbench
- Версионирование: Git + GitHub

Структура репозитория
В корне проекта находится файл README с описанием работы. Папка sql содержит основной скрипт создания и наполнения схемы базы данных schema.sql. Папка queries хранит четыре отдельных файла с учебными запросами, каждый из которых решает конкретную бизнес-задачу согласно варианту. В папке docs расположен отчёт о выполнении работы и подпапка images со скриншотами результатов выполнения запросов. Папка diagrams предназначена для размещения ER-диаграммы. Файл .gitignore исключает из репозитория системные, временные и конфиденциальные файлы.

Развёртывание базы данных
1. Склонируйте репозиторий:
2. Откройте phpMyAdmin или консоль MySQL.
3. Создайте новую базу данных с кодировкой utf8mb4.
4. Импортируйте файл sql/schema.sql через вкладку «Импорт» или выполните его содержимое.
5. База готова к работе. Проверьте таблицы через SHOW TABLES.

Архитектура уровня доступа к данным

Выбранный паттерн: Repository
Паттерн **Repository** был выбран для реализации DAL, так как он:
- Инкапсулирует логику доступа к данным в отдельных классах
- Предоставляет коллекционный интерфейс для работы с сущностями
- Отделяет бизнес-логику от логики работы с БД
- Упрощает тестирование и замену источника данных

Реализованные методы репозиториев

AbstractRepository (базовый класс)

| Метод | Назначение | Параметры |
|-------|------------|-----------|
| `findAll()` | Выборка всех записей с фильтрацией | `$where` (массив условий), `$orderBy` (сортировка), `$limit` (лимит) |
| `findById($id)` | Поиск по первичному ключу | `$id` (идентификатор) |
| `insert($data)` | Вставка новой записи | `$data` (ассоциативный массив) |
| `update($id, $data)` | Обновление записи | `$id`, `$data` (поля для обновления) |
| `delete($id)` | Удаление записи | `$id` (идентификатор) |

 LessonRepository (специфичные методы)

| Метод | Назначение | Пример использования |
|-------|------------|---------------------|
| `getLessonsByDate($date)` | Получить все уроки на дату | Расписание на день |
| `getTeacherLoad($teacherId, $start, $end)` | Нагрузка преподавателя за период | Аналитика загруженности |
| `createLessonWithValidation(...)` | Создание урока с проверкой бизнес-правил | Запись ученика с транзакцией |

 Пример вызова
```php
$lessons = new LessonRepository($pdo);
$todayLessons = $lessons->getLessonsByDate('2026-05-20');
$teacherLoad = $lessons->getTeacherLoad(1, '2026-05-01', '2026-05-31');
$newLessonId = $lessons->createLessonWithValidation(2, 3, 3, '2026-06-01 15:00:00', 60);
