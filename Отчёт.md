Цель
Реализовать CRUD-интерфейсы для управления справочными данными (Ученики, Преподаватели, Инструменты) с использованием MVC-архитектуры и ранее созданного DAL.

Архитектура
-Model: репозитории (`*Repository`) — инкапсулируют SQL
-View: PHP-шаблоны в `views/` — только отображение, экранирование через `htmlspecialchars()`
-Controller: классы в `controllers/` — обработка запросов, валидация, вызов репозиториев
-Router: `index.php` — простая маршрутизация через `?entity=...&action=...`

Безопасность
1. XSS: весь вывод экранируется `htmlspecialchars()`
2. CSRF: токены генерируются в `Validator::generateCsrfToken()`, проверяются при POST
3. SQL-инъекции: все запросы через подготовленные выражения в репозиториях
4. Валидация: серверная (обязательная) + клиентская (HTML5/JS, опционально)

Реализованные функции
| Сущность      | Страницы                         | Особенности                                                  |
|---------------|----------------------------------|--------------------------------------------------------------|
| Ученики       | list, create, edit, delete, view | Поиск по ФИО, пагинация 10/стр, проверка связей при удалении |
| Преподаватели | list, create, edit, delete, view | Валидация телефона, отображение инструментов через JOIN      |
| Инструменты   | list, create, edit, delete, view | Уникальность названия, подсчёт связанных уроков              |

Вывод
Интерфейс полностью соответствует требованиям: разделён на MVC-компоненты, использует DAL, защищён от основных уязвимостей, адаптивен и удобен для управления справочниками музыкальной школы.

Контрольный вопрос:
Вопрос: Зачем экранировать вывод через htmlspecialchars()?

Чтобы предотвратить XSS-атаки. Если злоумышленник введёт в поле <script>alert('xss')</script>, без экранирования этот код выполнится в браузере следующего пользователя. htmlspecialchars() преобразует < в &lt;, делая код безвредным текстом
